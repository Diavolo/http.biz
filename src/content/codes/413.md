El servidor se niega a procesar una solicitud porque el contenido de la solicitud es más grande de lo que el servidor está dispuesto o es capaz de procesar.

El servidor PUEDE cerrar la conexión para prevenir que el cliente continúe la solicitud.

Si la condición es temporal, el servidor DEBERÍA generar un campo de encabezado Retry-After para indicar que es temporal y después de qué tiempo el cliente PUEDE intentar de nuevo.
