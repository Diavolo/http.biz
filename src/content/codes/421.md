La solicitud fue dirigida a un servidor que no es capaz de producir una respuesta. Esto puede ser enviado por un servidor que no está configurado para producir respuestas para la combinación de esquema y autoridad que están incluidos en la URI de solicitud.

Los clientes que reciben una respuesta 421 Misdirected Request de un servidor PUEDEN reintentar la solicitud -- ya sea que el método de solicitud sea idempotente o no -- a través de una conexión diferente. Esto es posible si una conexión es reutilizada<sup>[1](#ref-1)</sup>o si se selecciona un servicio alternativo [ALT-SVC][3].

Este código de estado NO DEBE ser generado por proxies.

Una respuesta 421 es cacheable por defecto, es decir, a menos que se indique lo contrario por la definición del método o controles de caché explícitos<sup>[2](#ref-2)</sup>.
