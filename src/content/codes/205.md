El servidor ha cumplido con la solicitud y desea que el agente de usuario reinicie la "vista del documento", que causó que se enviara la solicitud, a su estado original como fue recibido del servidor de origen.

Esta respuesta está destinada a soportar un caso de uso común de entrada de datos donde el usuario recibe contenido que soporta entrada de datos (un formulario, bloc de notas, lienzo, etc.), ingresa o manipula datos en ese espacio, causa que los datos ingresados sean enviados en una solicitud, y luego el mecanismo de entrada de datos se reinicia para la siguiente entrada para que el usuario pueda fácilmente iniciar otra acción de entrada.

Dado que el código de estado 205 implica que no se proporcionará contenido adicional, un servidor NO DEBE generar contenido en una respuesta 205. En otras palabras, un servidor DEBE hacer una de las siguientes cosas para una respuesta 205: a) indicar un cuerpo de longitud cero para la respuesta incluyendo un campo de encabezado Content-Length con un valor de 0; b) indicar un contenido de longitud cero para la respuesta incluyendo un campo de encabezado Transfer-Encoding con un valor de chunked y un cuerpo de mensaje que consiste en un solo chunk de longitud cero; o, c) cerrar la conexión inmediatamente después de enviar la línea en blanco que termina la sección de encabezados.
